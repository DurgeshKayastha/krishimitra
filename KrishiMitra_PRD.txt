1.  Project Overview 

1.1  Product Name 

KrishiMitra — Your Smart Farming Companion 

1.2  One-Line Description 

A React web application that empowers Indian farmers with real-time crop prices, AI-driven crop 
recommendations, plant disease detection, and community alerts — all powered by free government 
APIs and Groq AI. 

1.3  Problem Statement 

Indian farmers, especially in rural Maharashtra (Kopargaon region), lack access to: 

• Real-time mandi prices near them, forcing reliance on middlemen 

• Data-driven crop selection based on their soil and local weather 

• Early disease detection before it spreads across multiple farms 

• A centralised platform combining all these tools in one place 

 

1.4  Target Users 

User Type Who They Are Primary Need 

Farmer (Primary) Small/medium landholders, Prices, crop suggestions, disease 
Kopargaon belt detection 

Farmer Leader / Panchayat Village-level coordinator Disease outbreak reports, alerts 

Agricultural Officer Krishi Vibhag officials View flagged disease reports in 
dashboard 

Developer / Admin You (Durgesh) Firebase admin, deploy, maintain

2.2  Page: Crop Prices Dashboard  (/prices) 

THIS IS THE MOST IMPORTANT PAGE. Separate full-page dashboard. Farmers check this daily. 

Features: 

• Auto-detect user's state using geolocation (browser navigator.geolocation), fall back to manual 
selector 

• Filter bar: State selector, District selector, Commodity selector, Date picker 

• Data table with columns: Commodity | Market | Min Price | Max Price | Modal Price | Date 

• Recharts line chart showing price trend for selected commodity over past 7 days 

• "Nearby Mandis" card: shows closest 3-5 mandis to the user based on GPS 

• Groq-powered price insight card: summarises whether prices are high/low this week in plain 
language 

• Export to CSV button 

API Integration: 

◦ Primary: data.gov.in API — GET https://api.data.gov.in/resource/9ef84268-d588-465a-a308-
a864a43d0070 (Agmarknet dataset). Needs free API key from data.gov.in. Register at data.gov.in to 
get key. 

◦ Parameters: api-key, format=json, filters[state]=Maharashtra, filters[district]=Ahmednagar, limit=50 



◦ IMPORTANT: This API does not support CORS. All calls MUST go through the Node.js backend 
proxy, not directly from React. 

◦ Groq call: Send the raw price data to Groq and ask it to generate a 2-3 sentence farmer-friendly 
insight. Example prompt in backend: 'Given these mandi prices for [commodity] in [district]: [data], 
write a 2-sentence insight for a farmer about whether to sell now or wait. Write in simple English.' 

Notes for Amazon Q: 

◦ Table must use shadcn/ui Table component 
◦ Add loading skeletons (shadcn/ui Skeleton) while data fetches 
◦ Handle empty state gracefully with a message and retry button 
◦ Mobile: hide some columns, show essential ones only 

 

2.3  Page: Crop Advisor  (/crop-advisor) 

Helps farmers decide WHAT to plant based on their location, soil type, and season. 

Features: 

• Step 1 — Location: User either allows GPS or manually enters state/district/taluka 

• Step 2 — Soil Input: Dropdown for soil type (Red, Black/Regur, Alluvial, Laterite, Sandy) + 
optional soil pH slider (4.0–9.0) + nitrogen/phosphorus/potassium dropdowns 
(Low/Medium/High) 

• Step 3 — Season: Kharif / Rabi / Zaid selector with current season auto-selected based on 
month 

• Step 4 — AI Suggestion: Sends all inputs to Groq and displays: 

◦ Top 5 recommended crops with brief reasoning 
◦ Best practices for each crop (irrigation, fertiliser, spacing) 
◦ Expected yield range per acre 
◦ Current market price trend for that crop (from Agmarknet) 

• Weather context card: Fetches from Open-Meteo and shows current temp, rainfall forecast, 
humidity 

• Soil Health Card link: Button linking to soilhealth.dac.gov.in for official soil report 

Groq Prompt Template (put this in backend): 

System: You are an expert agricultural advisor for Maharashtra, India. Always give practical, 
farmer-friendly advice. User: Location: {district}, {state}. Soil: {soilType}, pH {pH}, 
N:{nitrogen} P:{phosphorus} K:{potassium}. Season: {season}. Weather: Avg temp {temp}°C, 
Expected rainfall {rainfall}mm. Suggest top 5 crops. For each: crop name, why it suits this 
soil+climate, 3 best practices, expected yield per acre, current market outlook. 

API: 

◦ Open-Meteo: GET https://api.open-
meteo.com/v1/forecast?latitude={lat}&longitude={lon}&daily=temperature_2m_max,precipitation_sum
&timezone=Asia%2FKolkata&forecast_days=7 — NO API KEY NEEDED 

◦ Groq: use meta-llama/llama-4-scout-17b-16e-instruct model via backend 

 



2.4  Page: Disease Detector  (/disease-detect) 

Upload a photo of a diseased crop and get instant AI diagnosis + report to authorities. 

Features: 

• Image upload area: drag-and-drop or camera capture (mobile). Accept jpg/png/webp, max 5MB 

• Crop type selector (optional): helps improve accuracy — Tomato, Potato, Wheat, Rice, Cotton, 
Maize, Soybean, Sugarcane, Other 

• Detect Disease button: sends image to crop.health API 

• Results card showing: Disease name, Confidence %, Description, Affected parts 

• Groq Enhancement: Takes the raw API result and generates a farmer-friendly report with: What 
is this disease, How it spreads, Immediate actions, Organic remedy, Chemical remedy, 
Prevention 

• PDF Report download: Generates a formatted PDF with farmer details + diagnosis + Groq 
report 

• Alert Authorities button: Saves report to Firestore under /diseaseReports collection with: farmer 
name, contact, location (lat/lon), crop, disease, severity, image URL, timestamp. This creates an 
alert for agricultural officers. 

API Integration: 

◦ Primary: crop.health API (kindwise.com) — POST https://crop.health/api/v1/identification with base64 
image. Free tier: 100 identifications/month. Requires API key from kindwise.com (free signup). 

◦ Fallback: If crop.health fails or quota exceeded, send image to Groq vision model (meta-llama/llama-
4-scout-17b-16e-instruct supports vision) with prompt: 'Identify the plant disease in this image. Give: 
disease name, confidence, symptoms visible, immediate treatment.' 

◦ Firebase Storage: Upload image before analysis, store URL in Firestore report 

Notes for Amazon Q: 

◦ Show loading spinner with status messages: 'Analysing image...', 'Getting AI insights...' 
◦ Store recent detections in localStorage for history (last 5) 
◦ Alert button must show a confirmation dialog (shadcn/ui AlertDialog) before submitting 

 

2.5  Page: Weather & Farm Calendar  (/weather) 

Hyperlocal weather for the farmer's location with farming activity recommendations. 

Features: 

• Current weather card: temperature, humidity, wind speed, UV index, feels like 

• 7-day forecast strip with icons (sunny/rainy/cloudy) and daily high/low 

• Rainfall chart: Recharts bar chart of precipitation for next 7 days 

• Farming calendar: AI-generated suggestions like "Good day for sowing", "Avoid spraying 
pesticide (rain expected)", "Ideal for irrigation" 

• Weather alerts: If rainfall > 30mm predicted, show warning card in red 

API: 



◦ Open-Meteo: GET https://api.open-
meteo.com/v1/forecast?latitude=LAT&longitude=LON&hourly=temperature_2m,relative_humidity_2m,
windspeed_10m,uv_index&daily=temperature_2m_max,temperature_2m_min,precipitation_sum,weat
hercode&timezone=Asia%2FKolkata&forecast_days=7 

◦ NO API KEY REQUIRED for Open-Meteo 
◦ Groq: send 7-day forecast data, generate a farming calendar. Prompt: 'Given this 7-day weather 

forecast for a farmer in Maharashtra: {data}. Generate a day-by-day farming activity calendar with 1-2 
practical suggestions per day.'

5.  Firebase Setup Instructions 

🌾  DO THIS BEFORE GIVING PRD TO AMAZON Q. Complete all steps in this section first, then 
share the Firebase config values with Amazon Q. 

 

Step 1: Create Firebase Project 

• Go to console.firebase.google.com 

• Click Add project → Name it: krishimitra 

• Disable Google Analytics (not needed for now) 

• Click Create project 

 

Step 2: Enable Authentication 

• In Firebase Console → Build → Authentication → Get started 

• Enable Google sign-in provider 

• Enable Phone sign-in provider (for farmers without Google) 

• Set authorised domains: localhost, your-vercel-domain.vercel.app 

 

Step 3: Create Firestore Database 

• Build → Firestore Database → Create database 

• Start in TEST MODE (we will add rules later) 

• Choose region: asia-south1 (Mumbai — closest to Maharashtra) 

 

Step 4: Enable Storage 

• Build → Storage → Get started 

• Start in test mode 

• Region: asia-south1 

 

Step 5: Register Web App 

• Project Overview → Add app → Web (</> icon) 

• App nickname: KrishiMitra Web 

• DO NOT enable Firebase Hosting (we use Vercel) 

• Copy the firebaseConfig object — you will paste it into .env 

 



Step 6: Enable Google Maps API 

• Go to console.cloud.google.com (same Google account as Firebase) 

• Select your Firebase project from the dropdown 

• APIs & Services → Library → Search: Maps JavaScript API → Enable 

• APIs & Services → Library → Search: Geocoding API → Enable 

• APIs & Services → Credentials → Create Credentials → API Key 

• Restrict this key: Application restrictions → HTTP referrers → Add your localhost and Vercel 
domain 

• Restrict this key: API restrictions → Select Maps JavaScript API and Geocoding API only 

 

Step 7: Generate Service Account for Backend 

• Firebase Console → Project Settings (gear icon) → Service accounts 

• Click Generate new private key → Download JSON 

• Rename file to serviceAccountKey.json 

• Place in /backend/ folder 

• Add serviceAccountKey.json to .gitignore IMMEDIATELY 

 

Step 8: Firestore Security Rules 

Go to Firestore → Rules → Replace with this: 

rules_version = '2'; service cloud.firestore {   match /databases/{database}/documents {

10.  Key Instructions for Amazon Q 

🌾  Read this section first before starting any code. 

 

10.1  What Amazon Q Must NOT Do 

• Do NOT use class components. Use functional components with hooks only. 

• Do NOT use inline styles. Use Tailwind classes only. 

• Do NOT call government APIs (Agmarknet, crop.health) directly from React frontend. ALWAYS 
proxy through backend. 

• Do NOT hardcode any API key in any .jsx or .js frontend file. 

• Do NOT install paid packages or APIs without asking first. 

• Do NOT use any other CSS framework (no Bootstrap, no styled-components). 

• Do NOT change the colour scheme from Section 7 without asking. 

• Do NOT skip error handling. Every API call needs a try/catch and an error state. 

• Do NOT use TypeScript. JavaScript only (you are learning React, TypeScript adds complexity 
now). 

 

10.2  What Amazon Q MUST Do 

• Use React hooks: useState, useEffect, useContext, custom hooks in /hooks/ folder 

• Use React Router v6 for all navigation. Use <Link> not <a> for internal links. 

• Every page must have a loading state (shadcn/ui Skeleton) and an error state (shadcn/ui Alert) 

• Mobile responsive: test every component at 375px width minimum 

• Use shadcn/ui components as the base — do not build UI from scratch when shadcn has it 

• All Groq calls must include a clear system prompt that says: 'You are KrishiMitra, an AI assistant 
for Indian farmers. Always be helpful, practical, and specific to India. Keep responses concise 
and easy to understand.' 

• Use @tanstack/react-query (react-query) for all data fetching — it handles caching, loading 
states, and refetching automatically 

• All dates must be shown in DD/MM/YYYY format (Indian standard) 

• All prices must be shown in Indian number format (₹ symbol, lakhs/crores notation) 

 

10.3  When Amazon Q is Stuck 

If Amazon Q cannot find a data.gov.in API key or the crop.health API is unavailable, use these 
fallbacks: 

• Agmarknet fallback: Use static mock data for testing (5-10 sample records). Comment clearly: // 
MOCK DATA - Replace with real API call when data.gov.in key is available 

• Disease detection fallback: Use Groq vision model as primary and skip crop.health for now 



• Google Maps fallback: Use a simple Leaflet.js map (free, no key needed) as placeholder
